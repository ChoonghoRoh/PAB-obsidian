## 핵심 요약

공공 AI는 개별 기능 추가 사업이 아니라, 행정 운영체계를 데이터·AI·클라우드 기반으로 전환하는 전략 과제다. 행정안전부는 국정과제 체계에서 세계 최고 수준의 AI 민주정부 실현을 제시하고 있으며, 과학기술정보통신부는 AI 3대 강국 도약을 위한 AI 고속도로 구축, 세계에서 AI를 가장 잘 활용하는 국가 구현, AI 기본사회 실현 등을 주요 국정과제로 제시하고 있다. 이는 공공 AI 전환이 개별 기관의 정보화 사업을 넘어, 국가 차원의 행정 혁신과 디지털 기반 전환 과제라는 점을 시사한다.

정책 여건도 이미 실행 단계에 진입하였다. 정부는 중앙·지방정부가 내부망에서도 민간 AI 기술을 안전하게 활용할 수 있도록 범정부 AI 공통기반 서비스를 개시하였으며, 행정·공공 시스템을 클라우드 기반으로 전환하기 위한 로드맵 마련도 추진하고 있다. 따라서 현 단계에서 필요한 의사결정은 "AI를 도입할 것인가"가 아니라, 공통기반, 기관 전용 환경, 클라우드 운영 모델, 책임 체계를 어떻게 결합할 것인가에 관한 판단이다.

현시점에서는 범정부 공통기반을 우선 활용하고, 기관 특화 정확도와 보안 통제가 필요한 기능은 기관 전용 환경 또는 통제 강화형 환경으로 병행하는 방안을 우선 검토할 필요가 있다. 문서 요약, 공통 검색, 표준 API, 기본 질의응답처럼 여러 기관이 공통으로 활용할 수 있는 기능은 공통기반을 우선 활용하고, 내부 규정 검색, 기관별 민원 분류, 심사 기준 적용, 지역별 절차 안내처럼 기관 맥락이 정확도에 영향을 미치는 기능은 기관 전용 환경을 별도로 설계할 필요가 있다. 개인정보, 민감정보, 감사·심사 자료를 다루는 기능에는 별도의 통제 기준이 적용되어야 한다.

민관협력형 클라우드는 공공의 통제 기준을 유지하면서 운영 역량을 보완할 수 있는 운영 모델로 검토할 수 있다. 다만 이는 단순한 외주 확대가 아니라, 공공이 데이터 기준, 권한관리, 감사 대응, 서비스 우선순위를 주도하고 민간이 운영 자동화, 확장성, 장애 대응을 보완하는 방식이어야 한다. AI 에이전트는 전면 적용 대상이라기보다 규정 탐색, 초안 작성, 사례 비교, 협업 지원처럼 여러 업무 단계를 연결하는 시나리오형 고도화 기능으로 검토하는 것이 적절하다. 고성능 컴퓨팅은 대규모 분석, 영상·음성 처리, 연구·분석형 워크로드처럼 고연산 수요가 명확한 영역에서 별도로 검토할 필요가 있다.

### 의사결정자가 우선 승인해야 할 사항 (4가지)

- **첫째, 운영체계의 설정**: 공통기반 우선 활용과 기관 전용 환경 병행을 기본 원칙으로 채택할지 결정해야 한다.
- **둘째, 기능 배치 기준의 수립**: 범용 기능, 기관 특화 기능, 민감 기능, 고연산 기능을 어떤 기준으로 구분할지 정해야 한다.
- **셋째, 운영 책임체계의 명확화**: 데이터 최신성 관리, 오류 수정, 결과 검토, 장애 대응의 책임 주체를 사전에 정해야 한다.
- **넷째, 성과·비용 관리체계의 확정**: 기능 개수나 모델 성능이 아니라 업무시간 단축, 검색 정확도, 오류 수정, 사용자 재사용률, 검토·승인 이력, 운영비, 보안 대응비, 재해복구 비용까지 포함한 관리체계를 승인해야 한다.

이 네 가지는 정보화 부서만의 검토사항이 아니라, 현업 부서, 데이터 담당, 정보보호 담당, 기획·예산 부서가 함께 합의해야 할 기관 차원의 운영 기준이다.

## Ⅰ. 왜 지금 검토해야 하는가, 무엇을 선택해야 하는가

### 왜 지금 결정해야 하는가?

지금은 공공 AI의 가능성을 검토하는 단계가 아니다. 범정부 AI 공통기반이 내부망 활용을 전제로 개시되었고, 행정·공공 시스템의 클라우드 네이티브 전환도 정부 차원의 추진방향으로 제시되었다. 이는 공공 AI가 외부망 기반의 개별 실험이나 특정 부서의 도구 도입에 머물 수 없음을 의미한다. 앞으로의 핵심은 공공 AI 기능을 어떤 기반에 배치하고, 어떤 데이터와 권한 체계 아래 운영하며, 어떤 책임 구조로 지속시킬 것인가에 있다.

의사결정을 늦출 경우 기술 도입만 지연되는 것이 아니다. 데이터 정비, 권한관리, 업무 절차 재설계, 운영비 산정, 장애 대응 기준 수립도 함께 지연된다.

### 어떤 대안을 선택해야 하는가?

| 대안                  | 개념                                | 장점                        | 한계                  | 적합성       |
| ------------------- | --------------------------------- | ------------------------- | ------------------- | --------- |
| 공통기반 우선형            | 범용 기능을 범정부 공통기반 중심으로 도입           | 빠른 도입, 중복투자 감소, 확산 용이     | 기관 특화 정확도, 내부 통제 한계 | 제한적 적합    |
| 공통기반 + 기관 전용 환경 병행형 | 범용 기능은 공통기반, 특화 기능은 기관 전용 환경으로 병행 | 범용성과 정확도의 균형, 통제 수준 조정 가능 | 운영 책임 및 관리체계 설계 필요  | **우선 권고** |
| 기관 전용 환경 중심형        | 기관별 독자 구축·운영                      | 높은 통제 수준, 특화도 확보          | 구축비·운영비 부담, 재사용성 저하 | 선택적 검토    |

### 무엇을 기준으로 판단해야 하는가?

의사결정 기준은 기술 분류가 아니라 운영 분류여야 한다. 기관은 다음 여섯 가지를 우선 기준으로 삼을 필요가 있다:

1. 여러 기관이 공통으로 수행하는 기능인가?
2. 최근 개정 정보가 반영되지 않으면 오류 위험이 큰가?
3. 기관 내부 규정이나 고유 절차가 반영되지 않으면 정확도가 저하되는가?
4. 장애 발생 시 핵심 업무가 중단되는가?
5. 개인정보나 민감정보를 직접 처리하는가?
6. 일반 정보처리 자원보다 높은 연산 성능이 필요한가?

## Ⅱ. 운영체계 설계 방향

### 공통기반·기관 전용 환경·연계통제 계층

| 계층 구분 | 역할 | 적용 기능 |
|---|---|---|
| 공통기반 계층 | 범용 기능 (여러 기관 공통 활용) | 문서 요약·공통 검색·표준 API, 공통 로그·기본 질의응답 |
| 기관 전용 계층 | 기관 고유업무 정확도 확보 | 내부 규정 검색·기관별 민원 유형 분류, 사업 심사 기준·지역 사업 절차 안내 |
| 연계통제 계층 | 권한·보안·책임·장애 대응 | 개인정보·내부문서·심사·감사 자료 처리, 권한관리·로그·보안·예외처리 기준 |

**공통기반 계층**: 기관마다 별도로 만들 필요가 낮고, 범정부 차원의 표준화와 재사용 효과가 큰 기능을 우선 배치.

**기관 전용 계층**: 기관별 조례, 내부 지침, 과거 처리 사례, 사업별 예외 기준을 반영. 공통기반으로 처리하기 어려운 특화 영역에 한정.

**연계통제 계층**: 개인정보, 내부문서, 대외 발신, 심사·감사 자료가 포함되는 업무에서 접근 권한, 결과 근거, 오류·장애 책임을 관리.

### 민관협력형 클라우드 적용 원칙

| 공공의 역할 | 민간의 역할 |
|---|---|
| 데이터 기준·보안 요구·접근권한 설정 | 운영 자동화·확장성 확보·장애 대응 |
| 감사 대응·서비스 우선순위 결정 | 성능 관리·비용 최적화 |
| 기술 내재화 계획 수립·실행 | 계약 종료 후 데이터 반환·로그 보관 |

적용 범위는 공개·공통 기능, 운영 자동화, 장애 대응, 확장성 확보 영역부터 검토. 개인정보·감사·심사 자료 등 민감 정보가 포함되는 영역은 공공의 접근권한, 로그, 감사, 데이터 반환 기준을 먼저 확정한 뒤 판단.

### AI 에이전트 적용 원칙

**공공기관 우선 검토 가능 AI 에이전트 적용 시나리오**:
- 규정 탐색 후 보고서 초안 작성
- 민원 유형 분류 후 유사 사례 제시
- 심사 자료 검토 후 체크 항목 정리
- 회의 자료 요약 후 후속 조치 초안 작성
- 부서 간 협업 자료 정리와 이력 관리

※ 최종 판단·대외 발신·예외 사안 처리는 반드시 담당자의 검토·승인 필수

적용 여부 판단 기준: 기술 명칭이 아니라 해당 업무가 다단계 처리 구조를 갖고 있는지, 중간 결과의 검토·승인 지점이 명확한지, 로그와 책임소재를 남길 수 있는지.

### 고성능 컴퓨팅 적용 원칙

고성능 컴퓨팅은 다음과 같은 경우에 선별적으로 검토:
- 대규모 데이터 분석
- 실시간 추론
- 영상·음성 기반 처리
- 연구·분석형 워크로드
- 고성능 모델 개발·검증
- 대량 민원·재난·교통·환경 데이터 분석

일반적인 문서 요약, 규정 검색, 초안 작성, FAQ 응답 등은 우선 공통기반 또는 기관 전용 환경에서 처리.

## Ⅲ. 단계별 추진방향

### 단계별 추진 구조

공공 AI·클라우드 기반 구축은 **진단 → 정비 → 적용 → 확산 → 정착**의 단계로 추진.

| 단계                | 주요 과제                                        | 주요 확인사항                            |
| ----------------- | -------------------------------------------- | ---------------------------------- |
| 1단계: 기능·정보 자원 진단  | AI 적용 후보 기능, 참조 자료, 자료 보유 부서, 공통기반 활용 가능성 파악 | 기능 목록, 참조 자료 목록, 공통기반·기관 전용 후보 분류  |
| 2단계: 데이터·권한 기준 정비 | 자료 표준화, 접근권한, 로그·이력 관리, 오류 수정 절차 정리          | 데이터 준비 기준, 권한 기준, 로그·이력 관리 기준      |
| 3단계: 범용 기능 우선 적용  | 문서 요약, 검색, 초안 작성, 협업 보조 등 공통기반 적합 기능 적용      | 시범 적용 결과, 오류 유형, 사용자 피드백, 운영 역할 분장 |
| 4단계: 기관 전용 기능 확장  | 내부 규정, 민원 유형, 사업 절차가 반영되는 기능 확대              | 기관 전용 지식체계, 검토·승인 절차, 보안·재해복구 기준   |
| 5단계: 지속 운영체계 정착   | 운영비, 보안 대응비, 재해복구 비용, 성과지표 관리                | 운영비 산정표, 성과관리 지표, 개선·확산 계획         |

### 기관 유형별 적용 방향

| 기관 유형 | 우선 적용 방향 및 고려사항 |

## Ⅳ. 기반 구축을 위한 주요 검토사항

### 국정과제 정합성

- 공공부문 AI 활용 확대와 행정혁신 과제에 직접 연결되는가?
- 기관 단독 구축이 아니라, 범정부 확산체계와 정합적인가?
- 범정부 공통기반 재사용과 내부망 활용 방향에 부합하는가?
- 대국민 서비스 개선과 내부 업무혁신을 동시에 설명할 수 있는가?

### 범정부 AI 공통기반 정합성

- 본 기능은 범정부 AI 공통기반으로 우선 처리 가능한가?
- 기관 특화 기능은 공통기반 위에 추가 설계하는 방식이 가능한가?
- 공통기반 활용 시 중복 구축을 줄일 수 있는 범위는 어디까지인가?
- 내부망 활용, 데이터 접근, 보안 요구가 공통기반 운영 원칙과 충돌하지 않는가?

### 데이터 준비 기준

- 최근 개정된 자료가 검색 결과에 즉시 반영되는가?
- AI가 참조한 근거 자료를 확인할 수 있는가?
- 같은 주제의 민원, 보고서, 회의록, 지침이 함께 검색되는가?
- 오류 발견 시 수정 담당자와 절차가 명확한가?
- 자료별 열람 권한이 구분되어 있는가?
- 문서 변경 이력과 개정 이력이 관리되는가?

### 운영 환경 기준

기능별 위험 수준에 따라 일반 운영 환경, 기관 전용 환경, 통제 강화형 환경, 고성능 컴퓨팅 환경을 구분:

- 공개 안내·FAQ → 공통기반 또는 일반 운영 환경
- 내부 규정 검색 → 기관 전용 환경 필요
- 민원·복지 상담 → 통제 강화형 환경 (개인정보·로그 기준)
- 사업 심사·평가 지원 → 근거 추적과 책임 구조 별도 설계
- 영상·음성·연구 분석 → 고성능 컴퓨팅 수요 검토

### 민관협력형 클라우드 적용 기준

- 공공이 직접 통제해야 할 데이터·업무 범위가 명확한가?
- 민간이 담당할 수 있는 운영 자동화, 확장성, 장애 대응 범위가 구분되어 있는가?
- 장애 발생 시 보고·복구·책임 분담 절차가 정리되어 있는가?
- 계약 종료 후 데이터 반환, 로그 보관, 결과물 관리 기준이 마련되어 있는가?
- 보안 요구, 접근권한, 감사 대응 기준이 공공 주도로 설계되어 있는가?
- 공공 내부의 기술 내재화 계획과 운영 역량 확보 방안이 포함되어 있는가?

### AI 에이전트 적용 기준

- 적용 대상이 단일 검색이 아니라, 다단계 업무 지원 영역인가?
- 결과 검토, 승인, 로그 기록, 책임 소재가 함께 설계되어 있는가?
- 규정 탐색, 초안 작성, 사례 비교, 협업 지원 등 시나리오 단위로 시작할 수 있는가?
- 전면 확대가 아니라, 실증 가능한 업무 단위부터 적용할 수 있는가?

### 고성능 컴퓨팅 적용 기준

- 일반 행정 지원 기능이 아니라, 고연산 처리 수요를 동반하는가?
- 고성능 컴퓨팅 수요를 일반 운영 환경과 분리하여 예산·보안·운영 기준을 산정하였는가?
- 영상·음성 처리, 대규모 분석, 실시간 추론, 연구형 워크로드 여부가 명확한가?
- 상시 확보가 필요한지, 공동 활용 또는 제한적 활용으로 충분한지 판단하였는가?

### 성과관리 기준

- 성과지표가 기능 개수나 모델 성능에만 머물지 않는가?
- 문서 작성 시간 단축, 검색 정확도, 민원 처리 지원 효과를 측정하는가?
- 오류 수정 소요시간과 개선 이력을 관리하는가?
- 사용자 재사용률과 업무 적용률을 확인하는가?
- 검토·승인 이력과 책임성 지표를 포함하는가?
- 운영비, 보안 대응비, 재해복구 비용을 함께 관리하는가?

## Ⅴ. 정책적 시사점

### 3대 핵심 원칙

| ① 공통기반 우선 활용 원칙 | ② 기관 특화·민감 기능 분리 원칙 | ③ 운영 책임 선설계 원칙 |
|---|---|---|
| 범용 기능은 범정부 공통기반을 우선 활용하고, 기관별 중복 구축을 최소화한다. | 기관 고유 규정, 민원·심사 자료, 개인정보 처리 기능은 기관 전용 또는 통제 강화형 환경으로 분리한다. | 데이터 최신성, 오류 수정, 검토·승인, 장애 대응, 성과관리 책임을 기능 도입 전에 먼저 확정한다. |

### 4대 정책 방향

**첫째**: 공공 AI 정책은 공통기반의 재사용성과 기관별 정확도 요구 사이의 균형을 핵심 판단 기준으로 삼아야 한다.

**둘째**: 클라우드와 민관협력형 운영 모델은 기술 조달의 문제가 아니라 운영 책임의 문제로 다루어야 한다. 데이터 기준, 접근권한, 감사 대응, 장애 복구, 계약 종료 후 데이터 반환 기준은 공공이 주도해야 한다.

**셋째**: AI 에이전트와 고성능 컴퓨팅은 공공 AI 기반 구축의 고도화 수단으로 보되, 전면 적용보다 선별 적용이 필요하다.

**넷째**: 성과관리는 기능 수나 모델 성능 중심에서 실제 행정 성과와 책임성 중심으로 전환되어야 한다.

**결론**: 공공 AI·클라우드 기반 구축은 단일 시스템 구축 사업이 아니라 기관의 일하는 방식, 서비스 제공 방식, 데이터 관리 방식, 책임 구조를 함께 바꾸는 중장기 전환 과제다. 결국 공공 AI의 지속 가능성은 기술 성능보다 운영 책임과 통제 구조의 명확성에 의해 좌우된다.
