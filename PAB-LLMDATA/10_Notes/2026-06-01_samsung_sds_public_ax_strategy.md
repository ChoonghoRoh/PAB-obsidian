# 공공AX 전환 전략 — 공공 AI·클라우드 기반 구조 설계

## TL;DR
[원본 §핵심 요약 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#핵심-요약)

공공 AI 전환의 핵심 질문은 "도입할 것인가"가 아니라 **"공통기반, 기관 전용 환경, 클라우드 모델, 책임 체계를 어떻게 결합할 것인가"**다.

삼성SDS는 **공통기반 + 기관 전용 환경 병행형**을 우선 권고한다. 범정부 공통기반을 기본값으로 두되, 기관 특화 정확도·보안 통제가 필요한 기능만 기관 전용 또는 통제 강화형 환경으로 분리하는 전략이다.

핵심 원칙 3가지: ① 공통기반 우선 활용 → ② 기관 특화·민감 기능 분리 → ③ 운영 책임 선설계

## 왜 지금 결정해야 하는가
[원본 §왜 지금 결정해야 하는가? →](2026-06-01_samsung_sds_public_ax_strategy_source.md#왜-지금-결정해야-하는가)

- [[범정부 AI 공통기반]]이 내부망 활용 전제로 이미 개시됨 — 가능성 검토 단계 아님
- 의사결정 지연 시 기술 도입만 지연되는 것이 아니라, **데이터 정비·권한관리·운영비 산정·장애 대응 기준** 수립까지 함께 지연
- 지금의 의사결정은 단일 시스템 도입 여부가 아니라 **기관의 AI 운영체계 기준 결정**에 가까움

## 3대 운영 대안 비교
[원본 §어떤 대안을 선택해야 하는가? →](2026-06-01_samsung_sds_public_ax_strategy_source.md#어떤-대안을-선택해야-하는가)

| 대안 | 장점 | 한계 | 권고 |
|---|---|---|---|
| 공통기반 우선형 | 빠른 도입, 중복투자 감소 | 기관 특화 정확도 한계 | 제한적 적합 |
| **공통기반 + 기관 전용 병행형** | 범용성·정확도 균형, 통제 수준 조정 가능 | 운영 책임 설계 필요 | **우선 권고** |
| 기관 전용 환경 중심형 | 높은 통제·특화도 | 구축비·운영비 부담 | 선택적 검토 |

**기능 배치 판단 6대 기준** (기술 분류가 아닌 운영 분류):
1. 여러 기관이 공통으로 수행하는 기능인가?
2. 최근 개정 정보 미반영 시 오류 위험이 큰가?
3. 기관 내부 규정·고유 절차 미반영 시 정확도 저하되는가?
4. 장애 발생 시 핵심 업무가 중단되는가?
5. 개인정보·민감정보를 직접 처리하는가?
6. 일반 정보처리 자원보다 높은 연산 성능이 필요한가?

## 운영체계 3계층 구조
[원본 §공통기반·기관 전용 환경·연계통제 계층 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#공통기반기관-전용-환경연계통제-계층)

```
┌────────────────────────────────────────────┐
│  연계통제 계층 (권한·보안·책임·장애 대응)         │
│  개인정보·심사·감사 자료, 권한관리·로그·예외처리   │
├────────────────────────────────────────────┤
│  기관 전용 계층 (기관 고유업무 정확도)            │
│  내부 규정 검색, 민원 분류, 심사 기준, 지역 절차  │
├────────────────────────────────────────────┤
│  공통기반 계층 (범용 기능, 여러 기관 공통 활용)    │
│  문서 요약·공통 검색·표준 API·기본 질의응답       │
└────────────────────────────────────────────┘
```

- **공통기반 계층**: 중복 구축 감소, 범정부 표준화·재사용 효과 극대화
- **기관 전용 계층**: 조례·내부 지침·사업별 예외 기준 반영. 단, 모든 기능을 기관 전용으로 확대하면 중복 구축·운영비 증가 발생
- **연계통제 계층**: 공공 AI 운영의 안정성과 책임성을 확보하는 핵심 기반. 세 계층 중 가장 중요한 거버넌스 레이어

## 민관협력형 클라우드 역할 분담
[원본 §민관협력형 클라우드 적용 원칙 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#민관협력형-클라우드-적용-원칙)

| 공공의 역할 | 민간의 역할 |
|---|---|
| 데이터 기준·보안 요구·접근권한 설정 | 운영 자동화·확장성 확보·장애 대응 |
| 감사 대응·서비스 우선순위 결정 | 성능 관리·비용 최적화 |
| **기술 내재화 계획 수립·실행** | 계약 종료 후 데이터 반환·로그 보관 |

핵심 경고: 역할 구분이 불명확하면 민관협력형 클라우드는 **운영 효율화가 아니라 책임 분산**으로 이어진다. 민감 정보 포함 영역은 공공의 접근권한·로그·감사·데이터 반환 기준 먼저 확정 후 판단.

## AI 에이전트 적용 원칙
[원본 §AI 에이전트 적용 원칙 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#ai-에이전트-적용-원칙)

[[AI 에이전트]]는 전면 적용 대상이 아닌 **시나리오 단위 선별 적용** 대상:

**공공기관 우선 검토 시나리오**:
- 규정 탐색 후 보고서 초안 작성
- 민원 유형 분류 후 유사 사례 제시
- 심사 자료 검토 후 체크 항목 정리
- 회의 자료 요약 후 후속 조치 초안 작성
- 부서 간 협업 자료 정리와 이력 관리

**적용 판단 기준**: 기술 명칭 × → 다단계 처리 구조 여부, 검토·승인 지점 명확성, 로그·책임소재 가능성 ○

⚠️ 최종 판단·대외 발신·예외 사안 처리는 **반드시 담당자 검토·승인** 필수

## 고성능 컴퓨팅 선별 적용
[원본 §고성능 컴퓨팅 적용 원칙 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#고성능-컴퓨팅-적용-원칙)

모든 AI 기능에 GPU/NPU가 필요한 것이 아님. 다음 경우에만 별도 검토:

| 고연산 수요 명확 (GPU 검토 대상) | 일반 환경으로 충분 |
|---|---|
| 대규모 데이터 분석 | 문서 요약 |
| 실시간 추론 | 규정 검색 |
| 영상·음성 기반 처리 | 초안 작성 |
| 연구·분석형 워크로드 | FAQ 응답 |
| 대량 민원·재난·교통·환경 데이터 분석 | 공통 검색 |

고성능 컴퓨팅을 AI 도입비에 일괄 포함하기보다, **고연산 기능만 별도 예산·보안·운영 기준으로 분리** 검토.

## 5단계 추진 로드맵
[원본 §단계별 추진 구조 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#단계별-추진-구조)

```
1단계: 기능·정보 자원 진단
   → AI 적용 후보 기능, 참조 자료, 공통기반 활용 가능성 파악

2단계: 데이터·권한 기준 정비  ← 반드시 선행 (AI 도입 전 최소 요건)
   → 자료 표준화, 접근권한, 로그·이력 관리, 오류 수정 절차

3단계: 범용 기능 우선 적용
   → 문서 요약, 검색, 초안 작성, 협업 보조

4단계: 기관 전용 기능 확장
   → 내부 규정, 민원 유형, 사업 절차 반영 기능

5단계: 지속 운영체계 정착
   → 운영비·보안 대응비·재해복구 비용·성과지표 관리
```

**핵심 교훈**: 기능 먼저 도입 후 운영 기준 뒤따라 정비하는 방식은 실제 업무 활용 단계에서 중단 위험이 높다.

## 기관 유형별 우선 적용 방향
[원본 §기관 유형별 적용 방향 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#기관-유형별-적용-방향)

| 기관 유형 | 핵심 업무 | 우선 고려사항 |
|---|---|---|
| 중앙행정기관 | 정책·법령·협의 자료 등 지식집약형 | 근거 추적·대외 발신 승인 기준 우선 정립 |
| 지방자치단체 | 민원 응대·복지·지역 현안 (주민 접점 높음) | 조례·지역 맥락 자료 분리 운영 권장 |
| 공공기관 | 상담·신청·심사·정산·감사 (운영형) | 외부 접점 ↔ 내부 심사 기능 구분, 데이터 반환·로그 기준 필요 |
| 연구·교육기관 | 연구 검색·분석·모델링·고성능 연산 혼재 | 범용 공통기반 충분 여부 + 고성능 컴퓨팅 필요성 함께 검토 |

## 성과관리 기준 — 기능 수 → 실제 행정 성과
[원본 §성과관리 기준 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#성과관리-기준)

**기존 (잘못된 방향)**: 기능 개수, 모델 성능, 응답 속도

**올바른 성과지표**:
- 문서 작성 시간 단축
- 검색 정확도 향상
- 민원 처리 지원 효과
- 사용자 재사용률·업무 적용률
- 검토·승인 이력 (책임성 지표)
- 오류 수정 소요시간·개선 이력
- **운영비 + 보안 대응비 + 재해복구 비용** (총비용 관리)

## 의사결정자 체크리스트 (4대 승인 사항)
[원본 §의사결정자가 우선 승인해야 할 사항 →](2026-06-01_samsung_sds_public_ax_strategy_source.md#의사결정자가-우선-승인해야-할-사항-4가지)

- [ ] **운영체계 설정**: 공통기반 우선 활용 + 기관 전용 병행을 기본 원칙으로 채택?
- [ ] **기능 배치 기준**: 범용·특화·민감·고연산 기능 구분 기준 확정?
- [ ] **운영 책임체계**: 데이터 최신성·오류 수정·결과 검토·장애 대응 책임 주체 확정?
- [ ] **성과·비용 관리체계**: 업무시간 단축·검색 정확도·운영비·보안 대응비·재해복구 비용 포함 관리체계 승인?

> 이 네 가지는 정보화 부서만의 사항이 아니라, 현업·데이터·정보보호·기획예산 부서 **공동 합의** 필요.  
> 이 기준이 정리되지 않으면 공공 AI는 시범사업으로만 추진되고, 기관의 운영 방향으로 정착되기 어렵다.
