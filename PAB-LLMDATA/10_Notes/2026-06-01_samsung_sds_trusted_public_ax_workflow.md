# 신뢰 기반 공공AX 업무 체계 고도화 전략

## TL;DR
[원본 §핵심 요약 →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#핵심-요약)

공공AX가 시범을 넘어 현업에 정착하려면, **AI 기능 확대보다 [[업무관리 플랫폼]] 중심 운영체계 설계**가 핵심이다.

업무관리 플랫폼은 단순 협업 도구가 아니라 **AI 활용·접근권한·로그·기록관리·검토 이력·보안 정책이 함께 적용되는 공공AX의 핵심 운영 거점**. 접근법은 **전면 자동화 ✕ → 위험 기반 단계적 자동화 ○**. 보안·개인정보는 사후 보완이 아니라 운영 설계의 출발점.

> 시리즈 3부작 중 거버넌스·실행 편. 자매 보고서:
> - [[2026-06-01_samsung_sds_public_ax_strategy|공공AX 전환 전략]] (운영체계 3계층 구조)
> - [[2026-06-01_samsung_sds_public_ax_direction|공공AX 추진 방향 및 우선 과제]] (우선 업무·로드맵)

## 왜 업무 체계 고도화인가
[원본 §왜 업무 체계 고도화로 이어져야 하는가 →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#-공공ax는-왜-업무-체계-고도화로-이어져야-하는가)

공공기관은 이미 업무망·전자결재·민원·예산·인사·계약 시스템이 복잡하게 연결됨. AI를 단순 챗봇으로 추가하면 **결과 저장 위치·참조 자료·확인 절차 관리가 불가능**. 따라서 권한·로그·검토 절차·장애 대응·성과관리를 **업무 흐름 안에 내재화**해야 함. 핵심은 기술 도입이 아니라 업무관리 플랫폼·조직 운영 방식의 고도화.

## 업무관리 플랫폼 5계층 운영체계
[원본 §AI 기능을 업무관리 플랫폼 안에 어떻게 배치할 것인가 →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#-ai-기능을-업무관리-플랫폼-안에-어떻게-배치할-것인가)

```
⑤ 보안·거버넌스 기준  ← 전 과정에 함께 적용 (마지막 단계 ✕)
   권한·로그·검토·개인정보 영향평가·입력 지시문 관리·성과관리
④ 데이터·지식 기반
   내부 규정·법령·민원 사례·보고서·회의록·통계 (검색 가능)
③ AI 활용 기능
   요약·초안·규정 탐색·유사 사례 검색·분류·추천·AI 에이전트
② 업무 프로세스
   접수·분류·검토·확인·통지·기록 + 담당자 배정·처리기한·상태 추적
① 협업 접점
   플랫폼·문서·메일·메신저·회의·협업문서함·전자결재·민원 화면
```

**설계 4원칙**: ① 기존 업무 흐름 안에서 작동 ② 업무 위험도별 AI 활용 수준 구분 ③ 근거·처리 이력 기록 ④ 소규모 후 단계적 확산.

## 사례 — 경기도교육청 G-ONE
[원본 §사례 경기도교육청 G-ONE →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#사례--경기도교육청-g-one-업무관리-플랫폼-고도화)

삼성SDS 수행. 생성형 AI 대화형 서비스 + AI 문서 생성 + 업무협업 + 데이터 분석을 통합한 업무협업 플랫폼. 메일·메신저·회의·드라이브·문서·결재에 AI 결합 → 별도 시스템 이동 없이 회의록 작성·자료 검색·메일 초안·문서 요약·후속 조치 정리.

**시사점**: ① 협업툴 교체 ✕ → 운영체계 전환 ○ ② 현장 반복 병목 해소 시 수용성 확보 ③ 보안은 UX 반대가 아니라 안전한 협업 확장의 기반 조건 (통제된 클라우드·전용회선·통합 로그인).

## AI 에이전트 — 제한적·보조적 적용
[원본 §AI 에이전트 기반 업무 지원 →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#ai-에이전트-기반-업무-지원)

여러 업무 단계 연결 가능하나 **반복적·검토 가능한 업무에 제한적 적용**. 반복 워크플로우는 **도구 허용목록 + 단계별 확인 + 실행 로그 + 중단·복구 기준** 마련 후 제한적 적용.

외부 발신·파일 삭제·권한 변경·민감정보 조회·시스템 설정 변경은 **담당자 사전 검토를 기본값**으로. AI 에이전트는 공무원 판단 대체 ✕ → 정해진 범위 내 업무 단계 연결·담당자 확인 지원 보조 체계 ○.

## 보안·거버넌스 설계
[원본 §보안 및 거버넌스 기준은 어떻게 설계할 것인가 →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#-보안-및-거버넌스-기준은-어떻게-설계할-것인가)

### 신원 기반 접근관리 ([[제로트러스트]])
"내부망이면 안전하다" 전제 폐기 → 사용자·단말·서비스·데이터 흐름별 권한 세분화. AI 기능별 권한 차등, 고위험 기능 2인 확인, AI 에이전트 도구 호출 범위 제한, API 허용목록.

### 개인정보 영향평가
마스킹·입력 금지만으로 불충분. **입력·검색·요약·생성·저장 단계 중 어디서 개인정보가 처리되는지** 업무 설계 단계에서 검토. 민원·복지·인사·감사·계약은 더 엄격하게.

### AI 보안 위협 5종
| 위협 | 대응 |
|---|---|
| 지시 우회([[프롬프트 인젝션]]) | 시스템 지시-사용자 입력 분리, 출력 검토 |
| 데이터 유출 | 권한 기반 검색, 민감정보 탐지, 발신 전 확인 |
| 환각·근거 오류 | 검색 기반 근거 연결, 최신성 표시, 담당자 검토 |
| 도구 오남용 | 허용목록, 작업별 확인, 고위험 제한, 실행 이력 |
| 외부 모델·도구 위험 | 등록·버전 관리, 취약점 점검 |

### AI 거버넌스 8역할
**AI 운영위원회** 중심으로 역할 분리 — 적용 업무 추진 기능과 점검 기능을 구분 (동일 조직 단독 판단 시 내부 점검 약화):

서비스 담당 · 업무 프로세스 담당 · 데이터 관리 주체 · 보안 담당 · 개인정보 담당 · AI 운영 관리자 · 현업 검토자

> 거버넌스의 핵심은 특정 부서에 판단을 집중시키는 것이 아니라, 업무 적용·데이터·보안·개인정보·현업 검토가 **각자 역할에 따라 연결**되도록 하는 것.

## 5단계 실행 방안
[원본 §단계적 실행 방안 →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#-단계적-실행-방안)

```
1단계 운영 기준 수립        → AI 활용·제한 업무, 자동화 수준, 검토 기준, 역할 분담
2단계 플랫폼·지식 정비       → 업무 접점 연계, 협업문서함, 규정·사례 메타데이터화
3단계 범용 업무 적용         → 요약·정리·탐색·초안·분류·FAQ (초기 성과)
4단계 프로세스 자동화 확대   → 접수~사후기록 흐름 (민원·심사·계약·정책)
5단계 AI 에이전트·운영 고도화 → 다단계 지원, 모니터링, 성과 대시보드
```

**성과관리 5영역**: 업무 효율 · 품질·정확도 · 협업 · 보안·운영 투명성 · 운영 지속성. 초기엔 작성시간 단축·검색 정확도·만족도 등 체감 지표 중심, 확대 시 검토 이력·오류 개선·보안·운영비 추가.

## 5대 정착 방향 (결론)
[원본 §공공AX 정착 방향 →](2026-06-01_samsung_sds_trusted_public_ax_workflow_source.md#-공공ax-정착-방향-5대-결론)

1. **업무관리 플랫폼 고도화** — 현업 정착의 핵심 기반
2. **업무 프로세스 자동화** — 단위 기능 ✕ → 전체 흐름 ○
3. **AI 활용 + 담당자 검토 결합** — 신뢰 확보
4. **보안·개인정보·거버넌스** — 확산의 선행 조건
5. **성과관리** — 도입 실적 ✕ → 업무 개선 효과·운영 투명성 ○

> 성패는 AI 기능의 수가 아니라, **안전하게 재사용할 수 있는 업무 절차와 관리 기준을 기관 운영 역량으로 정착**시킬 수 있는가에 달려 있다.
