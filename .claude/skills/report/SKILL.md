# report -- 보고서 .md 파일 저장

## 역할

현재 대화에서 생성된 분석 결과, 비교 리포트, 설계 문서 등 보고서 성격의 출력물을 .md 파일로 저장한다.

## 입력

`$ARGUMENTS` -- 저장 경로 또는 파일명 (선택)

- 인수가 있으면: 해당 경로에 저장
  - 파일명만 전달 시: 현재 작업 디렉토리에 저장 (예: `report.md`)
  - 경로 포함 시: 해당 경로에 저장 (예: `docs/my-report.md`)
- 인수가 없으면: 보고서 내용을 기반으로 적절한 파일명을 자동 생성하여 현재 작업 디렉토리에 저장

## 실행 절차

### 1. 대상 콘텐츠 식별

- 현재 대화에서 가장 최근에 생성된 보고서/분석 출력을 식별한다.
- 표, 코드 블록, 구조화된 분석 등 보고서 성격의 출력을 대상으로 한다.

### 2. 파일명 결정

- `$ARGUMENTS`가 있으면 그대로 사용한다.
- 없으면 보고서 제목이나 내용에서 파일명을 추출한다.
  - 예: "ver5-1 vs ver-harness-sample 비교 분석" → `ver5-1-vs-harness-comparison-report.md`
  - 한글 제목은 영문 kebab-case로 변환한다.
- 확장자 `.md`가 없으면 자동 추가한다.

### 3. 파일 저장

- Write 도구로 .md 파일을 생성한다.
- 보고서 상단에 메타 정보를 포함한다:
  - 작성일 (오늘 날짜)
  - 원본 제목

### 4. 결과 보고

- 저장된 파일 경로를 사용자에게 알린다.
- 파일 크기(줄 수)를 함께 표시한다.
